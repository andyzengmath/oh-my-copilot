## 2. Approach

**Verbatim port + rename pass** (matches v0.1.x M1 pattern). For each skill:

1. Read OMX `skills/<name>/SKILL.md`
2. Apply text-level rewrites (rules below)
3. Write to OMGHC `skills/<name>/SKILL.md`
4. `npm run sync:plugin` mirrors to `plugins/oh-my-ghcopilot/skills/<name>/SKILL.md`

**Rename rules:**

- `omx` → `omghc`, `Omx` → `Omghc`, `OMX` → `OMGHC`
- `oh-my-codex` → `oh-my-ghcopilot`
- `~/.codex/` → `~/.copilot/`, `.codex/` → `.copilot/`
- Codex CLI references → Copilot CLI references *except* when describing the cross-CLI worker variant (preserve `codex` in those contexts; OMGHC's `tmux-session.ts:88` already supports codex/claude/gemini/copilot)
- `OPENAI_API_KEY` → Copilot auth env precedence (`COPILOT_GITHUB_TOKEN` > `GH_TOKEN` > `GITHUB_TOKEN`) — only in auth-mention contexts
- `omx <subcommand>` (CLI references) → either `omghc <subcommand>` (if shipped) or skill-only routing (if not)

**Key structural decisions:**

- **No new CLI subcommands.** All 14 surface via `$keyword` activation in Copilot sessions, exactly like v0.1.x ralph/team/etc. CLI parity for `ultrawork`/`ultraqa` deferred to v0.2.x if real users ask for it. (OMX itself moved `autoresearch` away from CLI.)
- **No new agent role prompts.** None of the 14 skills introduce new roles; the existing 33 prompts cover all referenced agent personas.
- **No `src/state/operations.ts` change.** `SUPPORTED_MODES` already includes `autoresearch`, `ultrawork`, `ultraqa` (added in M2 commit `065626a`).

## 3. Wiring changes

### Files modified (small)

1. **`src/hooks/keyword-detector.ts`** — add 14 keyword registrations (`$swarm`, `$review`, `$ralph-init`, `$ecomode`, `$deepsearch`, `$trace`, `$security-review`, `$ask-claude`, `$ask-gemini`, `$configure-notifications`, `$skill`, `$ultrawork`, `$ultraqa`, `$autoresearch`). Forward-compat — hooks don't fire in v1.0.40 (R-hooks-not-wired); detection actually happens via instructions.md routing today.

2. **`templates/instructions.md.tmpl`** — append a routing line per keyword inside the existing `<!-- OMGHC:SKILL-ACTIVATION:START -->` marker block.

3. **`src/catalog/__tests__/reader.test.ts`** — bump skill count assertion 21 → 35.

### Files NOT touched (auto-handled)

- `src/state/operations.ts` — `SUPPORTED_MODES` already complete.
- `src/cli/index.ts` — no new subcommands.
- `package.json` deps — no new dependencies; version bump 0.1.1 → 0.2.0 at release time only.
- `agents/*.agent.md` — no new role prompts needed.

### Existing infrastructure that auto-handles new skills

- `src/catalog/reader.ts` — iterates `skills/` directory.

- `npm run sync:plugin` — mirrors `skills/` → `plugins/oh-my-ghcopilot/skills/` automatically.
- `npm run verify:plugin-bundle` — checks parity at prepack time.
